# IOC和DI区别

（1）IOC: 控制反转，把对象创建交给spring进行配置——（解决对象的创建问题）
（2）DI: 依赖注入，向类里面的属性中设置值——（解决类之间依赖关系的问题）
（3）关系：依赖注入不能单独存在，需要在ioc基础之上完成操作

感悟：技术的出现都是为了解决某一个问题而存在。所以在学习某个技术的时候，最好能找到当初引起这个技术出现的问题，顺着这个问题去理解技术的设计思想。
